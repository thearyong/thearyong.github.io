github-page域名配置CNAME和MX记录冲突问题

发表于 2015-06-03   |     |   阅读次数

搭建博客配置域名时，添加 CNAME 记录到 thearyong.github.io，一直提示：

与其他记录冲突，不能添加
仔细查看发现，是跟 MX 记录冲突。MX 记录是之前配置域名邮箱添加的，不能删除。
MX 与 CNAME 不能共存的原因，是因为：
CNAME会影响MX的值，进而影响您域名邮箱的邮件收发。所以各大提供商控制后台都做了限制

所以 MX 和 CNAME 的 @ 记录是不能共存的。

那有没有其他办法？我只想要博客能正常访问，邮箱正常收发。

首先想到的方案是，@ 改成 A 记录。但是这样有个问题，如果 thearyong.github.io 后面 ip 变化了，A 记录需要重新配置。

后来发现github帮助中，有这样的描述：

之前在 thearyong.github.io—>settings 中配置的自定义域名是 thearyong.com，我们是不可以配置成 www.thearyong.com，
只要在域名解析配置 www 的即可。当输入 thearyong.com，github 帮我们重定向到 www.thearyong.com。

验证：

  1. thearyong.github.io 配置 www.thearyong.com 域名
  2. 域名解析后台 配置 CNAME www 记录 到 thearyong.github.io ，CNAME @ 记录不需要

最后，完美解决了邮箱收发和博客域名解析问题。
